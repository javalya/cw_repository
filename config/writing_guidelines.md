# 公众号写作规范

## 核心原则

### 1. 真实感与现场感
- **聊天记录截图分工**：
  - MickJagger 负责在文章中标记需要截图的位置（插入占位提示）
  - 刘总负责截取相关聊天记录并替换占位符
- 让读者感受到内容的真实性和即时性
- 避免过度包装，保持自然对话感

**占位符格式示例：**
```
📱【聊天记录截图位置】
- 场景：讨论XXX问题
- 建议截取：关于XXX的具体对话
- 目的：体现真实协作过程
```

### 2. 作者署名
- **统一使用笔名：MickJagger**
- 真实姓名仅在内部沟通使用，不出现在对外内容中

### 3. 文风优化（差评风格）

**口语化表达（按情绪场景分类）**

| 场景 | 示例 | 用法 |
|------|------|------|
| 惊讶/崩溃 | 我炸了、离谱、这谁顶得住、麻了 | 遇到意外问题时 |
| 吐槽/不满 | 就硬吹、蚌埠住了、什么鬼、服了 | 对现象或产品有意见 |
| 认同/顿悟 | 确实、说白了、真香了、懂了 | 点明核心观点或转折 |
| 质疑/反思 | 真的假的、我怀疑、等等，不对 | 引发思考或反转 |
| 轻松/调侃 | 有一说一、有一讲一、笑死 | 缓和语气，增加亲和 |

- 避免固定句式，根据内容自然选择
- 避免过于书面化的表达

**有情绪有态度**
- 表达真实感受，但**开头不要固定套路**
- 根据文章主题灵活开场：可以抛问题、讲冲突、直接上结论
- 不回避争议，敢于表达鲜明观点
- 犀利直接，不绕弯子

**故事感强（多种串联方式）**

方式一：时间节点（适合叙事类）
- 01 早上9点、02 十分钟后...

方式二：认知反转（适合技术揭秘类）
- 表象 → 深挖 → 破局 → 复盘

方式三：问题-解决（适合方案分享类）
- 遇到的坑 → 踩过的雷 → 找到的通路

方式四：情绪递进（适合体验分享类）
- 期待 → 失望 → 转机 → 真香

- 选择最适合文章内容的串联方式
- 让读者有跟随感，像在听亲身经历

### 4. 排版优化

**段落结构**
- 小段落，大量留白
- 避免大段文字堆砌

**视觉元素**
- 用emoji和符号突出重点：📋 📝 🔹 ✅
- 小标题更生活化（如"我炸了"、"真香了"）
- 引用格式增强对话感

**CSS样式规范**
```html
<section style="font-size: 16px; line-height: 1.8;">
  <!-- 段落 -->
  <p style="margin: 20px 0;">内容</p>
  
  <!-- 小标题 -->
  <h3 style="font-size: 20px; font-weight: bold; margin: 30px 0 15px 0;">小标题</h3>
  
  <!-- 引用块 -->
  <blockquote style="background: #f5f5f5; border-left: 4px solid #999; padding: 10px 15px; margin: 20px 0;">
    引用内容
  </blockquote>
  
  <!-- 列表项 -->
  <li style="margin: 10px 0;">列表内容</li>
  
  <!-- 结尾 -->
  <hr style="margin: 30px 0;">
  <p style="color: #999; font-size: 14px;">结尾小字</p>
</section>
```

### 5. 内容结构
- 开头要有吸引力，快速抓住读者注意力
- 中间内容翔实，有具体案例或数据支撑
- 结尾可以适当留白或引发思考

### 6. 配图要求
- 聊天记录截图要清晰完整
- 配图要与内容相关，增强真实感

### 7. 发布流程
1. 内容撰写完成（含截图占位提示）
2. 刘总截取聊天记录并替换占位符
3. 按规范进行排版优化
4. 检查笔名使用是否统一
5. 检查文风是否符合"差评风格"
6. 发布
7. 跟踪反馈，持续优化
