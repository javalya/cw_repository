服务器运维太麻烦？我用 Claude Code 搞定了 LanceDB 记忆系统搭建

MickJagger | 2026年3月12日

上一篇讲了为什么选择云服务器部署 OpenClaw。但上了云之后，第一个问题来了：AI 助手记不住我说过的话。折腾了 3 天，换了 3 种记忆方案，最后发现——服务器运维太麻烦了，直到我把 Claude Code 当成了 DevOps 助手。

一、默认记忆的痛点：AI 为什么总是"失忆"？

刚开始用 OpenClaw 的时候，我觉得默认的 memory-core 应该够用了。原理听起来也简单：

memory-core 原理：
• 用 Markdown 文件存储记忆片段
• SQLite 做索引加速查询
• 提供 memory_search 和 memory_store 两个工具函数

但用了几天，发现问题很严重：

1. 没有自动注入上下文
每次新会话，AI 完全不知道之前聊过什么。我得手动告诉它"去搜索记忆"，它才会去查。

2. 依赖模型的"自觉性"
memory-core 需要模型主动调用 memory_store 来保存重要信息。但问题是——模型经常"忘记"要存。聊完一整个会话，什么都没记下来。

3. 语义搜索能力弱
基于关键词匹配，而不是语义理解。我问"我之前说的邮箱是什么"，它可能搜不到，因为我当时说的是"我的邮箱是..."

核心痛点：跨会话失忆。昨天聊的内容，今天完全没印象。这不是我想要的"长期记忆"。

二、三种记忆方案对比：为什么最终选了 LanceDB？

为了解决记忆问题，我调研了市面上主流的三种方案，做了一个深度对比：

方案一：Mem0

优点：云端服务成熟、自动捕获记忆、生态完善

缺点：
• 需要稳定的国际网络（国内服务器访问不稳定）
• API 调用有成本
• 配置相对复杂，需要 API Key

我的评估：❌ 放弃
国内服务器网络环境不稳定，不想依赖外部 API。

方案二：QMD（Memory-Core 升级版）

优点：OpenClaw 原生支持、纯本地、透明可编辑

缺点：
• 仍然需要手动调用 memory_search
• 模型得"记得"保存信息
• 内存占用较高（对我的 2核4G 服务器来说可能吃力）

我的评估：❌ 放弃
2核4G 配置跑 QMD 可能会内存吃紧，而且自动记忆的问题没解决。

方案三：LanceDB ✅

优点：
• Auto-Recall：自动注入相关记忆到上下文
• Auto-Capture：自动捕获并保存重要信息
• 向量语义搜索：理解语义，不只是关键词匹配
• 官方唯一认可：OpenClaw 唯一内置推荐的三方向量库
• 内存可控：适合 2核4G 配置（我的服务器刚好是这个配置）

缺点：配置复杂、需要本地部署 Embedding 模型

我的评估：✅ 选择
虽然配置麻烦，但功能完全符合需求，而且官方认可。

选择 LanceDB 的两大核心原因

原因 1：服务器配置限制
我的服务器只有 2核4G，跑 QMD 可能会内存吃紧。LanceDB 更轻量，向量检索效率高，适合小配置服务器。

原因 2：官方唯一认可
翻看 OpenClaw 官方文档，LanceDB 是唯一被内置推荐的三方向量数据库。这意味着更好的兼容性和长期维护保障。

三、最终方案落地：Docker + Ollama + LanceDB

确定了 LanceDB 方案，下一步是技术选型。我需要解决一个问题：国内服务器无法下载 HuggingFace 的 Embedding 模型。

技术栈架构

┌─────────────────────────────────────────┐
│ OpenClaw Gateway │
│ (18789端口) │
└─────────────────┬───────────────────────┘
│
┌─────────────────▼───────────────────────┐
│ LanceDB Memory Plugin │

└─────────────────┬───────────────────────┘
│
┌─────────────────▼───────────────────────┐
│ Ollama (Docker) │
│ (nomic-embed-text模型) │
│ - 文本向量化 (768维) │

└─────────────────────────────────────────┘

为什么选择 nomic-embed-text？

• 轻量级：768 维向量，适合 2核4G 配置
• 本地部署：通过 Ollama 运行，无需外网下载
• 开源免费：无 API 调用成本
• 效果够用：与 LanceDB 配合，语义检索效果良好

部署步骤（理想版 vs 现实版）

理想版（你以为的）：
1. 安装 LanceDB 插件
2. 配置 Ollama
3. 重启 OpenClaw
4. 记忆功能正常
耗时：10 分钟

5. 手动改代码，vim 太难用
6. 不同环境命令不适配（Ubuntu vs CentOS）
