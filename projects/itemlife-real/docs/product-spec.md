## 2. 产品方案

### 2.1 产品定位
**一句话**: 极简物品寿命管理工具，帮你做出更理性的消费决策

**核心价值**:
- 知道每件物品用了多久
- 知道什么时候该换了
- 知道每天为这件物品付多少钱

### 2.2 目标用户
**主要用户**: 极简主义者
- 物品数量少但每件都很重要
- 关注物品的使用价值
- 理性消费，不冲动购物

**次要用户**: 
- 租房族（管理有限物品）
- 数码爱好者（追踪设备寿命）
- 家庭主妇（管理家电保修期）

### 2.3 MVP功能清单

#### P0 - 核心功能（必须）
| 功能 | 说明 |
|------|------|
| 添加物品 | 名称、分类、购买日期、价格、预期寿命 |
| 物品列表 | 按剩余寿命排序，寿命条可视化 |
| 物品详情 | 使用年限、剩余天数、日均成本 |
| 本地存储 | 数据保存在本地，保护隐私 |

#### P1 - 重要功能（优先）
| 功能 | 说明 |
|------|------|
| 寿命提醒 | 提前7天/30天通知 |
| 统计面板 | 物品总数、总花费、即将到期 |
| 分类管理 | 数码/衣物/家具等分类 |
| 深色模式 | 系统适配 |

#### P2 - 增值功能（后续）
| 功能 | 说明 |
|------|------|
| 云同步 | Firebase备份（可选） |
| 数据导出 | CSV备份 |
| Widget小组件 | 桌面显示即将到期物品 |

#### ❌ 明确不做（MVP阶段）
- 社交功能
- 扫码录入
- AI识别物品
- 多人共享
- 商城推荐

### 2.4 差异化策略
| 维度 | 我们 | 竞品 |
|------|------|------|
| 速度 | 3秒添加一件 | Excel需填很多字段 |
| 专注 | 只做物品寿命 | 随手记功能太杂 |
| 体验 | 极简无干扰 | Notion学习成本高 |
| 隐私 | 本地优先 | 很多App强制登录 |
