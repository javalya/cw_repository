## 产品矩阵

### App 1: 情绪急救室 (Mental Health)
**定位**: 职场人3分钟快速平复情绪工具
**差异化**: 不做大而全的冥想，只做"急救"场景

| 维度 | 详情 |
|------|------|
| **目标用户** | 25-35岁职场人，高压工作，情绪急救需求 |
| **核心功能** | 1. 情绪快速评估(3个问题)<br>2. 3-5分钟急救方案(呼吸/冥想/认知重构)<br>3. 情绪日记(极简记录)<br>4. 压力预警(连续高压提醒) |
| **技术栈** | Flutter + Firebase + 本地AI模型 |
| **变现** | 免费基础功能 + ¥18/月高级方案 |
| **市场规模** | 全球心理健康App $143.7亿，年增16% |
| **MVP周期** | 3周 |

**核心卖点**: 
- 快：打开App到开始急救<10秒
- 轻：每次使用<5分钟
- 隐：不社交、不分享、纯私人

### App 2: 手帐Pro (Hand Account/Journaling)
**定位**: 数字手帐工具，专为手帐爱好者设计
**差异化**: 比GoodNotes轻量，比备忘录专业

| 维度 | 详情 |
|------|------|
| **目标用户** | 手帐爱好者、学生、精致生活人群 |
| **核心功能** | 1. 多层级笔记本管理<br>2. 模板库(周计划/月计划/打卡)<br>3. 手写识别+搜索<br>4. 贴纸/胶带素材库<br>5. 导出分享(PDF/图片) |
| **技术栈** | Flutter + 自研手写引擎 + 云同步 |
| **变现** | 免费基础版 + ¥30/年解锁模板素材 |
| **市场规模** | 数字笔记App用户亿级，手帐细分垂直 |
| **MVP周期** | 4-5周 |

**核心卖点**:
- 专业：手帐专用功能(打卡追踪/习惯统计)
- 自由：手写+打字混合
- 美：高颜值模板，手帐感

**刘总自用价值**: 可以作为日常规划工具，同时测试迭代

### App 3: 物品寿命 (Item Lifespan)
**定位**: 极简主义者的物品管理工具
**差异化**: 不是记账，是记录"东西的生命"

| 维度 | 详情 |
|------|------|
| **目标用户** | 极简主义者、断舍离践行者、整理控 |
| **核心功能** | 1. 拍照记录物品+购买日期<br>2. 计算使用频次/天数<br>3. 寿命预估+更换提醒<br>4. 丢弃记录+原因<br>5. 年度物品报告 |
| **技术栈** | Flutter + 本地存储 + OCR识别 |
| **变现** | 免费基础版 + ¥12/年高级统计 |
| **市场规模** | 小众但精准，极简主义趋势增长 |
| **MVP周期** | 3周 |

**核心卖点**:
- 反消费：让你知道买了多少没用的东西
- 数据化：物品使用可视化
- 极简：App本身也要极简

## 开发路线图

### Phase 1: MVP验证期 (第1-2个月)
**目标**: 快速上线第一个App，验证模式

| 周次 | 任务 | 产出 |
|------|------|------|
| W1 | 确定第一个App + 设计PRD | 产品文档 |
| W2-3 | MVP开发 | 可测试版本 |
| W4 | 内测+迭代 | 优化版本 |
| W5-6 | 上架准备+内容预热 | 素材/文案 |
| W7-8 | 上架运营+数据观察 | 上线版本 |
