# 需求调研 Agent (Needs Research Agent)

**版本**: 轻量级 v1.0
**功能**: 自动化发现用户痛点和App需求机会
**更新日期**: 2026-03-20

---

## 快速开始

### 安装依赖
```bash
pip install requests beautifulsoup4 schedule openai
```

### 配置
编辑 `config.yaml`:
```yaml
# 调研平台配置
platforms:
  - name: reddit
    enabled: true
    subreddits:
      - appideas
      - sideproject
      - entrepreneur
    keywords:
      - "app idea"
      - "looking for"
      - "need an app"
      - "wish there was"
  
  - name: xiaohongshu
    enabled: true
    keywords:
      - "求推荐"
      - "有没有app"
      - "想找个工具"
  
  - name: jike
    enabled: true
    topics:
      - 效率工具
      - 独立开发者

# 报告配置
report:
  frequency: weekly  # weekly/daily
  day: monday
  time: "10:00"
  min_mentions: 3    # 最少提及次数才收录

# 输出配置
output:
  format: markdown
  path: ./reports/
  webhook: ""  # 飞书/钉钉webhook，可选
```

### 运行
```bash
python needs_agent.py --config config.yaml
```

## 核心功能

### 1. 痛点爬虫 (Pain Point Crawler)
- 自动搜索多个平台的用户需求表达
- 提取关键词和上下文
- 去重和归一化处理

### 2. 频率统计 (Frequency Analyzer)
- 统计同一痛点的提及次数
- 识别趋势上升的需求
- 标记高优先级机会

### 3. 竞品扫描 (Competitor Scanner)
- 自动搜索现有解决方案
- 评估市场饱和度
- 找出差异化空间

### 4. 报告生成 (Report Generator)
- 自动生成周报
- 高亮高频痛点
- 提供可行性建议

## 自动化集成

### 方式1: OpenClaw Cron定时任务
```yaml
# openclaw_config.yaml
cron_jobs:
  - name: needs_research_weekly
    schedule: "0 10 * * 1"  # 每周一10点
    command: "python /workspace/skills/needs-research-agent/needs_agent.py --report weekly"
    notify: feishu
```

### 方式2: GitHub Actions
```yaml
# .github/workflows/needs-research.yml
name: Weekly Needs Research
on:
  schedule:
    - cron: '0 10 * * 1'
jobs:
  research:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Run Needs Agent
        run: python needs_agent.py --report weekly
      - name: Upload Report
        uses: actions/upload-artifact@v3
        with:
          name: needs-report
          path: reports/
```
