# INSIGHT AI STUDIO 内容生产流程（含审核机制）

## 流程概览

```
┌─────────────────────────────────────────────────────────────────┐
│                     内容生产全流程                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  12:00  ┌──────────┐     ┌──────────┐     ┌──────────┐        │
│  热点 ─→│ 内容策划  │────→│ 内容审核  │────→│ 刘总终审  │        │
│  监控   │  (撰写)   │     │  (审核)   │     │  (发布)   │        │
│         └──────────┘     └────┬─────┘     └──────────┘        │
│                               │                                 │
│                         不通过 ↓ 打回修改                       │
│                               │                                 │
└───────────────────────────────┴─────────────────────────────────┘
```

---

## 角色分工

### 1. 内容策划师（Content Lead）
- **职责**：监控热点、选题策划、撰写文章
- **输出**：文章初稿（HTML格式）
- **交付物**：文章文件 + 来源清单

### 2. 内容审核师（Content Auditor）
- **职责**：事实核查、来源验证、合规审查
- **输出**：审核报告（通过/不通过 + 修改意见）
- **交付物**：审核意见

### 3. 刘总（最终决策者）
- **职责**：终审、决策发布
- **输出**：发布指令
- **交付物**：发布确认

---

## 详细流程

### 阶段一：热点监控（每日12:00）

**执行者**：内容策划师
**工具**：定时任务 + kimi_search

1. 搜索当日科技热点
2. 筛选TOP5热点
3. 生成简报发送到飞书群
4. 推荐当晚发文选题

---

### 阶段二：内容撰写（12:00-18:00）

**执行者**：内容策划师
**输出**：文章初稿

**要求**：
- 每个关键事实标注来源
- 数据标注时间和出处
- 引用标注原始链接
- 不确定的内容标注【待核实】

**交付格式**：
```
文章标题：XXX
文件路径：/path/to/article.html

关键事实来源清单：
1. 事实：DeepSeek V3.2发布
   来源：https://api-docs.deepseek.com/
   核实状态：✅ 已核实
   
2. 事实：文远知行2000台订单
   来源：https://finance.sina.com.cn/...
   核实状态：⚠️ 单一来源，需补充
```

---

### 阶段三：内容审核（18:00-19:00）

**执行者**：内容审核师
**工具**：kimi_search + kimi_fetch

**审核步骤**：

1. **接收任务**
   ```
   审核请求：
   - 文章标题：XXX
   - 文件路径：XXX
   - 来源清单：XXX
   ```

2. **逐条核实**
   - 使用搜索工具验证每个关键事实
   - 访问原始来源确认信息
   - 交叉验证数据

3. **输出审核报告**
   
   **情况A：审核通过**
   ```
   ✅ 内容审核通过
   
   文章标题：DeepSeek V3.2：国产大模型的新进化
   审核时间：2026-03-09 18:30
   
   核实结果：
   - 关键事实5条，均已核实
   - 信息来源3个，均为官方或权威媒体
   - 无违规内容
   
   建议：可以提交给刘总终审
   ```
   
   **情况B：审核不通过**
   ```
   ❌ 内容审核未通过
   
   文章标题：XXX
   审核时间：2026-03-09 18:30
   
   问题列表：
   1. 🔴 【虚假事实】"DeepSeek V4已发布"
      涉及内容："DeepSeek V4在3月8日正式发布..."
      核实结果：DeepSeek V4尚未发布，目前最新版本为V3.2
      修改建议：改为"DeepSeek V3.2已发布"或删除相关内容
      
   2. 🟡 【来源单一】文远知行订单数据
      涉及内容："文远知行获得2000台订单"
      核实结果：仅找到一个来源
      修改建议：补充其他媒体报道或标注"据新浪财经报道"
   
   修改后请重新提交审核。
   ```

---

### 阶段四：修改完善（如需要）

**执行者**：内容策划师
**动作**：根据审核意见修改
**循环**：修改后重新提交审核

---

### 阶段五：刘总终审（19:00）

**执行者**：刘总
**输入**：审核通过的文章
**决策**：
- 通过 → 推送到公众号
- 不通过 → 说明原因，打回修改

---

## 质量把控机制

### 三重审核

| 层级 | 审核者 | 审核重点 | 通过率 |
|------|--------|---------|--------|
| 第一重 | 内容策划师 | 自我检查 | 90% |
| 第二重 | 内容审核师 | 事实核查 | 70% |
| 第三重 | 刘总 | 终审决策 | 100% |

### 审核红线

以下情况**绝对不允许通过**：
- 虚假事实（如编造产品发布）
- 无法验证的信息
- 侵权内容
- 违规内容

## 工具配置

### 定时任务设置

```json
// 12:00 热点监控
{
  "name": "科技热点监控-中午摘要",
  "schedule": "0 12 * * *",
  "sessionTarget": "isolated",
  "payload": {
    "kind": "agentTurn",
    "message": "【内容策划师】执行热点监控任务..."
  }
}

// 18:00 内容审核（由内容策划师触发）
// 19:00 终审发布（由刘总确认后执行）
```

### 工作流触发

当内容策划师完成文章后，自动触发：
```
sessions_spawn("内容审核师", "审核任务：...")
```
